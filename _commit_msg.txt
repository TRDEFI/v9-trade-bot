### 1. botRunner.ts — Mutex Race Condition Fix [CRITICAL]
PROBLEM: max_open=4 config but 9-12 positions opened simultaneously
Race condition: multiple signals arrive in same 500ms loop → all pass the
"length < max_open" check before any position is added to openPositions
FIX: Added openingPosition mutex flag

CHANGES:
+ openingPosition = false  // class property — mutex flag
+ if (this.openingPosition) { logToFile("[BOT] SKIP: Opening mutex locked"); }
+ this.openingPosition = true  // lock BEFORE checking again
  ... position opening logic ...
+ } finally { this.openingPosition = false; }  // always unlock

WHY: Opening lock is set BEFORE the max_open check, not after.
This guarantees only ONE position can be evaluated/opened per loop cycle,
even if multiple signals pass the initial check.

### 2. botRunner.ts — Margin Call Cooldown [CRITICAL]
PROBLEM: After margin call, bot closed worst position then immediately opened
new positions → death spiral
LOG EVIDENCE:
  [error] Kritik Kasa Zarar Limiti! ... kasayi rahatlatmak icin
  [info] OPEN LONG AXLUSDT @ 0.0598  ← immediately after!

FIX: marginCallCooldown = Date.now() + 5 * 60 * 1000 (5 minute cooldown)
Added to:
+ Class property: marginCallCooldown = 0
+ On margin call trigger: this.marginCallCooldown = Date.now() + 5 * 60 * 1000
+ At signal check start: if (Date.now() < this.marginCallCooldown) continue;

BEHAVIOR: If margin call fires, no new positions open for 5 minutes.
If a position closes via TP during cooldown → openCount drops but no new
position opens until cooldown expires. This breaks the death spiral.

### 4. strategy.ts — RSI Calculation Bug Fix
PROBLEM: When diff > 0 (price went up), avgLoss was only decayed with
(period-1)/period multiplier but NO new loss value was added.
This caused RSI to be skewed — avgLoss stayed artificially high when
there were consistent gains, making RSI appear more bearish than reality.

BEFORE:
  if (diff > 0) {
    avgGain = (avgGain * (period - 1) + diff) / period;
    avgLoss = (avgLoss * (period - 1)) / period;  // BUG: loss not updated!
  }

AFTER:
  const gain = diff > 0 ? diff : 0;
  const loss = diff < 0 ? -diff : 0;
  avgGain = (avgGain * (period - 1) + gain) / period;
  avgLoss = (avgLoss * (period - 1) + loss) / period;

Both avgGain and avgLoss are updated every iteration regardless of direction.

### 5. strategy.ts — EMA Seed Value Fix
PROBLEM: EMA was seeded with klines[0].c (first candle close).
This is non-standard and produces inaccurate EMA values early in the series.
Standard Wilder EMA uses SMA of first 'period' candles as seed.

AFTER:
